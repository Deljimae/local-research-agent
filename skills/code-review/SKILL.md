---
name: code-review
description: Use this skill when the user asks for a code review, bug risks, regressions, or missing test coverage.
---

When using this skill:
- Focus on findings first, not summaries.
- Prioritize correctness issues, regressions, and missing validation.
- Cite concrete files and line numbers when possible.
- Call out missing tests when they materially increase risk.
- If no major issues are found, say that explicitly and mention residual risk.
